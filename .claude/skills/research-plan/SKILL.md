# Research Plan

Synthesize existing context into an analysis plan. Do not overfit the plan to a
software product workflow.

## Process

1. Read `CONTEXT.md`, `data/README.md`, and relevant decisions.
2. Identify the research objective and the output the plan is meant to produce.
3. Draft the plan using the template below.
4. Ask the researcher to confirm scientific scope, data availability, and
   validation strategy before turning it into work packages.

## Template

```md
# Analysis Plan: <title>

## Research Objective

## Expected Output

Paper section, figure, table, dataset, model, map, report, or decision.

## Inputs

Datasets, notes, manuscripts, code, prior results, and access constraints.

## Method

The analytical approach and why it fits the question.

## Validation

How we will know the result is credible.

## Risks And Caveats

Data gaps, assumptions, confounders, sensitivity, reproducibility risks.

## Work Packages

High-level tasks, not yet issue-sized.

## Out Of Scope
```
